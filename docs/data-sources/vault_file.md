# ansible_vault_file (DataSource)

Decrypts an ansible-vault encrypted file and exposes the plaintext content as a sensitive computed attribute.

Only the decrypted `content` is stored in Terraform state. The `path` and `vault_password_file` attributes are read from configuration on every plan/apply and are never persisted to state. For workflows where even the decrypted content must not appear in state, use the [`ansible_vault_file` ephemeral resource](../ephemeral-resources/vault_file.md) instead (requires Terraform 1.10+).

## Example Usage
```terraform
data "ansible_vault_file" "secret" {
  path               = "${path.module}/secrets/db_password.yml"
  vault_password_file = "${path.module}/.vault_pass"
}

output "db_password_yaml" {
  value     = data.ansible_vault_file.secret.content
  sensitive = true
}
```
